# Real-World Multimodal Demo

This guide explains how to record a value-focused demo of the Nifty50 multimodal Transformer pipeline.

The purpose of the demo is not to claim trading performance. The purpose is to show, step by step, how real market data is converted into aligned multimodal embeddings and how the fusion model can be evaluated through modality ablations.

---

## Demo thesis

The core question is:

> Can a model gain a richer view of a stock when it sees market numbers, chart structure, text context, and peer/sector relationships together?

This project demonstrates that pipeline:

```text
yfinance OHLCV snapshots
  -> technical features
  -> future outperformance labels
  -> rolling tabular windows
  -> candlestick chart images
  -> image tokens
  -> market-summary text records
  -> text tokens
  -> lightweight KG context
  -> KG tokens
  -> aligned multimodal artifact
  -> FusionTransformer
  -> ablation results
```

---

## What is real versus conceptual

Real artifacts generated by the demo:

- real OHLCV/index data from yfinance snapshots;
- real feature and label rows in `tabular_samples.csv`;
- real candlestick chart PNGs in `charts/`;
- real aligned `tabular_tokens`, `image_tokens`, `text_tokens`, `kg_tokens`, and `y` arrays in the NPZ artifact;
- real one-epoch ablation metrics if `--run-ablations` is used.

Conceptual or lightweight parts:

- text records are deterministic market-summary text derived from real OHLCV features, not scraped news/filings;
- KG is lightweight sector/peer/event context, not a large financial knowledge graph;
- one-epoch ablation metrics are pipeline evidence, not investment-grade model validation;
- portfolio backtesting is not yet implemented as a full historical trading simulation.

Do not present generated conceptual slides as backtest results. Use them only to explain architecture.

## Step 1: Run the real-world demo

Quick artifact build:

```bash
python scripts/run_real_world_demo.py \
  --output-dir data/processed/real_world_demo \
  --period 9mo \
  --window-size 20 \
  --horizon-days 3
```

Build plus ablations:

```bash
python scripts/run_real_world_demo.py \
  --output-dir data/processed/real_world_demo \
  --period 9mo \
  --window-size 20 \
  --horizon-days 3 \
  --run-ablations \
  --epochs 1 \
  --batch-size 4 \
  --device cpu
```

Default universe:

```text
Stocks:
- RELIANCE.NS
- TCS.NS
- INFY.NS

Benchmark:
- ^NSEI
```

Override the universe directly from the command line:

```bash
python scripts/run_real_world_demo.py \
  --tickers RELIANCE.NS TCS.NS INFY.NS HDFCBANK.NS ICICIBANK.NS SBIN.NS \
  --benchmark ^NSEI \
  --output-dir data/processed/real_world_demo \
  --period 9mo \
  --window-size 20 \
  --horizon-days 3 \
  --run-ablations \
  --epochs 1 \
  --batch-size 4 \
  --device cpu
```

For a first run or screen recording, start with the default 3-stock universe. Once it works, rerun with 6-10 stocks for richer charts and ablation visuals.

## Step 3: Show how raw data becomes embeddings

The aligned artifact is:

```text
data/processed/real_world_demo/real_world_multimodal_samples.npz
```

Inspect it with:

```bash
python - <<'PY'
import numpy as np
path = 'data/processed/real_world_demo/real_world_multimodal_samples.npz'
data = np.load(path, allow_pickle=False)
for key in data.files:
    print(f'{key:16s}', data[key].shape, data[key].dtype)
PY
```

Expected keys:

```text
tabular_tokens
image_tokens
text_tokens
kg_tokens
y
end_dates
stock_ids
```

Narration:

> The important transformation is that heterogeneous data becomes vectors. Tabular windows become numeric token sequences. Chart PNGs are encoded through the image Transformer. Text records are converted into leakage-safe text vectors. KG context becomes numeric graph-context tokens. The fusion model sees these as aligned modality embeddings for the same stock and date.

## Step 7: What visualizations should be generated from actual data

For a strong recorded demo, use actual run artifacts rather than conceptual images wherever possible.

Recommended visuals to add or generate next:

1. **Embedding projection**  
   Project `tabular_tokens`, `image_tokens`, `text_tokens`, `kg_tokens`, or fused embeddings into 2D using PCA/t-SNE/UMAP. Show whether samples cluster by stock, label, or modality.

2. **Ablation comparison chart**  
   Plot `accuracy`, `precision`, `recall`, `f1`, and `roc_auc` from `ablation_results.csv`.

3. **Real candlestick examples**  
   Show 2-3 generated PNGs from `charts/` next to their `stock_id`, `end_date`, and label.

4. **Modality agreement table**  
   For each sample, compare scores from tabular-only, +KG, +image, +text, and all-modality variants once prediction export is added.

5. **Backtest curve**  
   This is not implemented yet. The next value feature is to convert ranked predictions into a simple top-k portfolio simulation versus Nifty and plot cumulative return, drawdown, and turnover.

## Recording script

Suggested 6-8 minute flow:

### 1. Problem framing

> Stock prediction is usually built from numbers alone. But a real analyst looks at multiple signals: price behavior, chart structure, recent narrative, sector context, and peer behavior. This project asks whether a Transformer can fuse these views into one representation.

### 2. Show raw inputs

Open the generated real-world demo folder and show:

```text
raw/
tabular_samples.csv
charts/
text_records.csv
event_records.csv
```

Say:

> These are real run artifacts. The only lightweight part is that the text is currently generated from market summaries, not scraped news.

### 3. Show alignment

Open `DEMO_SUMMARY.md` and explain:

```text
stock_id + end_date
```

Say:

> Alignment is what turns separate modules into a real multimodal dataset.

### 4. Show embeddings

Run the NPZ inspection command and show shapes.

Say:

> Every modality is now represented as a tensor. This is the bridge from raw multimodal data to embedding space.

### 5. Show fusion

Open `src/models/fusion.py` or explain from README:

> The fusion Transformer projects every modality into a common dimension, concatenates modality tokens, adds modality embeddings, and applies self-attention across modalities.

### 6. Show ablation results

Open `ablation_results.csv`.

Say:

> The right question is not whether the architecture looks multimodal. The right question is whether additional modalities change evaluation behavior. That is why we run ablations.

### 7. Be honest about backtesting

Say:

> We have classification/ablation evidence now. The next step is a portfolio-style backtest that turns ranked predictions into a top-k strategy and compares it against Nifty.
