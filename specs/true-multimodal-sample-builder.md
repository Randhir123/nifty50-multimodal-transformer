# True Multimodal Sample Builder

## Goal

Convert the project from a set of modality-specific modules into an aligned multimodal training pipeline.

A single model sample must represent one stock at one prediction date. Every modality attached to that sample must be available at or before that prediction date, while the label is computed from future relative return after that date.

## Sample contract

Each row represents:

```text
(stock_id, prediction_date)
```

For that row:

- tabular tokens come from the rolling OHLCV feature window ending at `prediction_date`.
- image tokens come from the candlestick chart for the same stock using rows where `date <= prediction_date`.
- text tokens come from company records where `event_date <= prediction_date`.
- KG tokens come from deterministic market graph context as of `prediction_date`.
- `y` is the binary label for future outperformance over the configured horizon.

## Output artifact

The builder writes one `.npz` artifact with this schema:

Required keys:

- `tabular_tokens`: `[num_samples, window_size, tabular_dim]`
- `y`: `[num_samples]`
- `end_dates`: `[num_samples]`
- `stock_ids`: `[num_samples]`

Optional keys:

- `image_tokens`: `[num_samples, image_dim]` or `[num_samples, seq, image_dim]`
- `text_tokens`: `[num_samples, text_dim]` or `[num_samples, seq, text_dim]`
- `kg_tokens`: `[num_samples, kg_dim]` or `[num_samples, seq, kg_dim]`

This directly matches the existing `src.training.train_fusion` loader contract.

## First implementation slice

This first slice is intentionally CPU-only and toy-data compatible. It adds:

1. a reusable array container for aligned multimodal samples;
2. validation that all modality arrays have identical sample counts;
3. a deterministic toy-data builder;
4. a leakage-safe text token helper that ignores future text records;
5. a deterministic KG-context-to-token helper;
6. a CLI wrapper that writes a toy `multimodal_samples.npz` artifact;
7. unit tests for schema, saving/loading, text cutoff, and KG feature stability.

## Non-goals for this slice

- no production text scraping;
- no OCR;
- no graph neural network;
- no large pretrained image/text encoders;
- no claim that non-tabular modalities improve metrics yet.

Those belong in later slices after the aligned artifact is stable.

## Acceptance criteria

The following commands should pass:

```bash
pytest tests/unit/test_multimodal_sample_builder.py
python scripts/build_multimodal_samples.py --toy-output data/processed/multimodal_samples.npz
python -m src.training.train_fusion \
  --dataset data/processed/multimodal_samples.npz \
  --use-image --use-text --use-kg \
  --epochs 1 --batch-size 4 --device cpu
```

## Later slices

After this slice, wire real modality encoders into the same schema:

1. tabular windows from `src.data.dataset.create_rolling_transformer_dataset`;
2. chart images generated by `src.viz.charts` and encoded by `ImageTransformer.encode_images`;
3. normalized text assembled by `src.data.text.build_company_text_input` and encoded by the text branch;
4. KG context from `src.kg.query_graph.retrieve_kg_context`;
5. an ablation runner comparing tabular-only vs multimodal variants.
