# Spec: Portfolio Backtester

## 1. Objective
Implement a historical portfolio backtest that converts a model's ranked outperformance predictions into a simple top-k trading strategy. Compare the strategy's performance against the Nifty50 benchmark (`^NSEI`).

## 2. Inputs and outputs
- **Inputs**:
  - `prediction_scores_<variant>.csv` (generated by ablation runs or inference script) containing `row_id`, `end_date`, `y_true`, `y_prob`.
  - Raw market data or `tabular_samples.csv` to retrieve actual forward returns for the stocks and the benchmark.
- **Outputs**:
  - `backtest_metrics.json` containing total return, annualized return, max drawdown, and turnover.
  - `backtest_curve.png` showing the cumulative return of the top-k strategy versus Nifty50.

## 3. Alignment and leakage rules
- **Strict No-Lookahead**: Capital allocation on day `T` must rely solely on predictions formed from data up to `T`.
- **Forward Returns**: The performance from `T` to `T + horizon_days` must be computed using true future prices, simulating an actual holding period.

## 4. Implementation boundaries
- **In scope**: Daily or period-based step-through of predictions. Selection of the top `k` stocks ranked by `y_prob`. Equal-weighting among selected stocks. Calculating portfolio return vs baseline.
- **Out of scope**: Transaction costs, slippage, short-selling, or complex risk-parity position sizing.

## 5. Testing strategy
- Unit test with a small toy dataframe (e.g., 3 days, 3 stocks) to manually verify that the selected top-k stocks' returns average correctly to the portfolio return.

## 6. Acceptance criteria
- The command `python scripts/run_backtest.py --predictions ... --top-k 1` generates a valid performance plot and metrics file.
- Documentation `docs/real-world-demo.md` is updated to include this final evaluation step.

## 7. Open questions
- If `horizon_days=3` (default), do we rebalance every 3 days, or do we rebalance daily and maintain overlapping 3-day holding periods? For the initial slice, a daily rebalance using the latest available prediction is standard and simplest to implement.
